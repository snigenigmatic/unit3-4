Why WordPiece Chose Rare Pairs First
High score occurs when count(xy) is close to count(x) × count(y). For rare characters like i, n, g — which co-occur only as "ing" — their joint probability is "perfect". WordPiece recognizes this structural signal even though raw frequency is low. This is why BERT's tokenizer captures meaningful morphemes ("ing", "ed", "tion") so well.
Problem T.3 Medium
SentencePiece — Space-as-Token Tokenization
SentencePiece treats whitespace as just another character via the special symbol ▁ (U+2581). This makes it language-agnostic.
Input Sentence
"GenAI changes everything"
Learned Vocabulary (after training)
{▁, Gen, AI, ▁change, s, ▁every, thing, e, v, r, y, t, h, i, n, g, a, ...}
Compute
(a) Rewrite the input with the ▁ marker
(b) Apply greedy longest-match tokenization
(c) Show how the tokenizer recovers the original string
(d) State two advantages of SentencePiece
(a) Rewrite with ▁ Marker

Every space becomes ▁, and (by convention) a leading ▁ is added to the first word too — to mark word boundary uniformly.

Original: "GenAI changes everything" Rewritten: "▁GenAI▁changes▁everything"
(b) Greedy Longest-Match Tokenization

Scan the rewritten string, picking the longest vocab token at each position.

"▁GenAI▁changes▁everything" Pos 0: Try '▁GenAI'? not in vocab. '▁Gen'? not in vocab. But 'Gen' is in vocab — we need '▁' as separate too. ▁ alone IS in vocab → take "▁" Pos 1: 'Gen' ✓ longest → take "Gen" Pos 4: 'AI' ✓ → take "AI" Pos 6: '▁' → take "▁" Pos 7: '▁change' starts here? text from pos 6 onward is "▁changes▁..." Restart: at pos 6 we have '▁change'? YES (7 chars) → take "▁change" Wait — better: scan greedy. Reset: Greedy scan: i=0: '▁' (longer '▁G', '▁Ge'... not in vocab) → "▁" i=1: 'Gen' (longest) → "Gen" i=4: 'AI' → "AI" i=6: '▁change' is 7 chars; check '▁changes'(8)? not in vocab. '▁change' YES → "▁change" i=13: 's' → "s" i=14: '▁every' YES → "▁every" i=20: 'thing' YES → "thing" Tokens: [▁, Gen, AI, ▁change, s, ▁every, thing]
(c) Detokenization (Round-trip)

To recover the original text:

1. Concatenate tokens: "▁GenAI▁changes▁everything" 2. Replace ▁ with space: " GenAI changes everything" 3. Strip leading space: "GenAI changes everything"

This lossless reversibility is a defining property of SentencePiece.

(d) Two Key Advantages
  • Language-agnostic: No pre-tokenization needed. Works on Japanese/Chinese (no spaces) the same way as English.
  • Lossless & reversible: Original raw text can be reconstructed from tokens (unlike whitespace tokenizers which collapse multiple spaces).
  • Bonus: Used by Llama, T5, mT5, NLLB — all multilingual models.
[▁, Gen, AI, ▁change, s, ▁every, thing] — 7 tokens, fully reversible

GPT-2/3/4, BERT, Llama, T5 — all use subword tokenization (BPE or its variants) because it provides the best balance: ~30-50K vocabulary, no OOV problem, and reasonable sequence lengths (~1.3 tokens per word for English).

Problem E.1 Hard
Full BLEU-2 Score with Brevity Penalty
Compute the full BLEU-2 score (combining 1-gram and 2-gram precisions with brevity penalty).
Reference & Candidate
Reference: "the quick brown fox jumps over the lazy dog" (r = 9) Candidate: "the quick fox jumps over lazy dog" (c = 7)
BLEU-N Formula
BLEU-N = BP · exp(Σn=1..N wn · log pn) = BP · (∏ pn)1/N
BP = 1 if c > r, else exp(1 − r/c)
pn = clipped n-gram matches / total candidate n-grams
Compute
(a) Clipped 1-gram precision p1
(b) Clipped 2-gram precision p2
(c) Brevity penalty BP
(d) Final BLEU-2 = BP · (p1·p2)1/2
(a) Unigram Precision p1

Candidate 1-grams: the, quick, fox, jumps, over, lazy, dog → 7 total

Reference counts: the=2, quick=1, brown=1, fox=1, jumps=1, over=1, lazy=1, dog=1

Clipped matches: min(cand_count, ref_count) per unique word:

the: min(1, 2) = 1 quick: min(1, 1) = 1 fox: min(1, 1) = 1 jumps: min(1, 1) = 1 over: min(1, 1) = 1 lazy: min(1, 1) = 1 dog: min(1, 1) = 1 Total clipped = 7 p₁ = 7/7 = 1.0000
(b) Bigram Precision p2

Reference bigrams (8 total): (the,quick), (quick,brown), (brown,fox), (fox,jumps), (jumps,over), (over,the), (the,lazy), (lazy,dog)

Candidate bigrams (6 total): (the,quick), (quick,fox), (fox,jumps), (jumps,over), (over,lazy), (lazy,dog)

Check each candidate bigram against reference:

(the, quick): ✓ in reference → match (quick, fox): ✗ not in reference ("quick brown" in ref) (fox, jumps): ✓ → match (jumps, over): ✓ → match (over, lazy): ✗ not in reference ("over the" in ref) (lazy, dog): ✓ → match Clipped matches = 4 p₂ = 4/6 = 0.6667
(c) Brevity Penalty

c = 7 (candidate length), r = 9 (reference length). Since c < r:

BP = exp(1 − r/c) = exp(1 − 9/7) = exp(−2/7)
−2/7 = −0.2857 BP = exp(−0.2857) = 0.7515

BP < 1 penalizes the candidate for being shorter than the reference.

(d) Final BLEU-2

Geometric mean of precisions (with N = 2):

(p₁ · p₂)^(1/2) = (1.0000 · 0.6667)^0.5 = √0.6667 = 0.8165 BLEU-2 = BP · 0.8165 = 0.7515 · 0.8165 = 0.6136
BLEU-2 = 0.6136
Interpretation
  • Perfect unigram precision (p₁ = 1.0) — all candidate words appear in reference.
  • Bigram precision drops because removed words break local word order ("quick brown" → "quick fox").
  • BP further penalizes the short candidate.
  • A score of ~0.6 is typical for usable but imperfect translations.
Problem E.2 Medium
Why Clipping Matters — Repetition Trap
A buggy translator outputs repeated tokens. Show how unclipped vs clipped precision differ.
Reference & Candidate
Reference: "the cat sat on the mat" (r = 6) Candidate: "the the the the the the" (c = 6, all "the")
Compute
(a) Naïve (unclipped) unigram precision
(b) Clipped unigram precision
(c) Brevity penalty
(d) BLEU-1 with clipping — why is clipping critical?
(a) Naïve Precision (BAD — no clipping)

"the" appears 6 times in candidate, and it does appear in reference, so a naïve count would say all 6 match:

Naïve matches = 6 Naïve precision = 6/6 = 1.0000 (!!)

This is clearly wrong — the candidate is nonsense. This is why clipping was introduced.

(b) Clipped Precision

"the" appears max 2 times in the reference. So clipped count is capped:

the: min(cand=6, ref_max=2) = 2 Total clipped matches = 2 p₁ = 2/6 = 0.3333
(c) Brevity Penalty

c = 6, r = 6. Since c = r (not less):

BP = 1 (no penalty, lengths match)

BLEU-1 = BP · p₁ = 1.0 · 0.3333 = 0.3333
Clipped BLEU = 0.333 (vs naïve 1.000) — clipping drops nonsense translations correctly

Without clipping, BLEU would reward repetition. Clipping ensures each reference token can contribute at most once per its occurrence in the reference — preventing gaming the metric with repeated common words.
Problem E.3 Medium
Brevity Penalty Edge Cases
Compute BP for three candidates of different lengths against the same reference.
Reference
Reference: "the team won the match yesterday" (r = 6) Candidate A: "team won yesterday" (c = 3) Candidate B: "the team won the match" (c = 5) Candidate C: "the team won the match yesterday in stadium" (c = 8)
Compute
BP for each candidate. Explain the asymmetric behavior.
BP Formula
BP = 1 if c > r BP = exp(1 − r/c) if c ≤ r
Candidate A (c = 3, r = 6)
c < r, so BP = exp(1 − 6/3) = exp(−1) = 0.3679 Severe penalty (37%) for being half the length.
Candidate B (c = 5, r = 6)
c < r, so BP = exp(1 − 6/5) = exp(−0.2) = 0.8187 Mild penalty (~82%) for being slightly shorter.
Candidate C (c = 8, r = 6)
c > r, so BP = 1.0 (no penalty) Longer candidates are NOT penalized by BP — but they get penalized through lower precision (extra words become "wrong" by clipping).
BP: A = 0.368, B = 0.819, C = 1.000 — penalty grows exponentially as candidate shrinks
Why Asymmetric?
  • Short candidates can have artificially high precision (fewer tokens to be wrong about) → BP corrects this.
  • Long candidates have more chances to add unmatched words → precision naturally drops, so no extra penalty needed.
  • The exponential form ensures small length mismatches are gentle but big ones are harsh.
